# Coverage Validation Script Documentation

## Overview

The coverage validation scripts validate backend code coverage against per-layer thresholds and report detailed coverage gaps when validation fails. Three versions are provided for different environments:

- **`validate-coverage.ps1`** - Windows/PowerShell version
- **`validate-coverage.sh`** - Linux/Mac shell version  
- **`validate-coverage.py`** - Python version (for CI/CD pipelines)

## Requirements Implemented

**Requirement 1.1: Backend Coverage Measured with Layer Thresholds**

Coverage Thresholds:
- **Domain Layer**: 85% minimum line coverage
- **Application Layer**: 82% minimum line coverage
- **Infrastructure Layer**: 78% minimum line coverage
- **API Layer**: 80% minimum line coverage

## Features

### Per-Layer Threshold Checking
- Validates each project layer against its specific threshold
- Fails validation if ANY layer falls below threshold
- Generates per-layer status reports

### Detailed Coverage Gap Reporting
- Identifies coverage gap size (% below threshold)
- Lists classes with lowest coverage (< 50%)
- Provides recommended remediation actions
- Saves detailed reports to file for CI/CD integration

### Coverage File Analysis
- Searches for `coverage.cobertura.xml` files generated by Coverlet
- Supports nested test result directories
- Uses most recent coverage file if multiple found
- Gracefully handles missing or invalid files

### Exit Codes for CI/CD
- **Exit 0**: All layers passed - validation successful
- **Exit 1**: One or more layers failed - validation failed

## Usage

### PowerShell Version (Windows)

**Basic Usage:**
```powershell
.\scripts\validate-coverage.ps1
```

**With Custom Directories:**
```powershell
.\scripts\validate-coverage.ps1 -TestResultsDir "c:\tests" -ReportOutputDir "c:\reports"
```

**In CI/CD Pipelines:**
```powershell
& ".\scripts\validate-coverage.ps1"
if ($LASTEXITCODE -ne 0) {
    Write-Host "Coverage validation failed"
    exit 1
}
```

### Shell Version (Linux/Mac)

**Basic Usage:**
```bash
./scripts/validate-coverage.sh
```

**With Custom Directories:**
```bash
./scripts/validate-coverage.sh --test-dir ./api-server/tests --report-dir ./coverage
```

**In CI/CD Pipelines:**
```bash
./scripts/validate-coverage.sh
if [ $? -ne 0 ]; then
    echo "Coverage validation failed"
    exit 1
fi
```

### Python Version (CI/CD)

```python
import subprocess
result = subprocess.run(["python3", "scripts/validate-coverage.py"], cwd=".")
if result.returncode != 0:
    print("Coverage validation failed")
    sys.exit(1)
```

## How It Works

### Coverage File Discovery

1. Script scans `api-server/tests/` directory (or custom test directory)
2. Searches for `coverage.cobertura.xml` files in each project's test directory
3. Uses most recent coverage file if multiple found
4. Supports nested `TestResults/` directories

### Coverage Extraction

1. Parses Cobertura XML to extract per-package coverage percentages
2. Converts decimal values (0.85) to percentages (85.0%)
3. Rounds to 2 decimal places for readability
4. Falls back to overall coverage if package-specific coverage not found

### Threshold Validation

1. Compares actual coverage against defined thresholds
2. Calculates coverage gap (threshold - actual)
3. Identifies classes with low coverage (< 50%)
4. Generates remediation recommendations

## Generating Coverage Files

### Generate Coverage for All Tests

```bash
# Windows
dotnet test --collect:"XPlat Code Coverage" --settings:coverlet.runsettings

# Linux/Mac
dotnet test --collect:"XPlat Code Coverage" --settings:coverlet.runsettings
```

### Generate Coverage for Specific Project

```bash
dotnet test api-server/tests/NaarNoor.Domain.Tests `
    --collect:"XPlat Code Coverage" `
    --settings:coverlet.runsettings
```

## Integration with CI/CD

### GitHub Actions Example

```yaml
name: Coverage Validation

on: [pull_request]

jobs:
  validate-coverage:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup .NET
        uses: actions/setup-dotnet@v3
        with:
          dotnet-version: '8.0.x'
      
      - name: Run tests with coverage
        run: |
          dotnet test `
            --collect:"XPlat Code Coverage" `
            --settings:coverlet.runsettings
      
      - name: Validate coverage
        run: |
          .\scripts\validate-coverage.ps1
          if ($LASTEXITCODE -ne 0) {
              exit 1
          }
      
      - name: Upload coverage report
        if: failure()
        uses: actions/upload-artifact@v3
        with:
          name: coverage-gap-report
          path: coverage/coverage-gap-report.txt
```

### Azure Pipelines Example

```yaml
trigger:
  - main

pool:
  vmImage: 'windows-latest'

steps:
- task: UseDotNet@2
  inputs:
    version: '8.0.x'

- task: DotNetCoreCLI@2
  displayName: 'Run tests with coverage'
  inputs:
    command: 'test'
    arguments: '--collect:"XPlat Code Coverage" --settings:coverlet.runsettings'

- task: PowerShell@2
  displayName: 'Validate coverage thresholds'
  inputs:
    targetType: 'inline'
    script: '.\scripts\validate-coverage.ps1'

- task: PublishBuildArtifacts@1
  displayName: 'Upload coverage report'
  condition: failed()
  inputs:
    PathtoPublish: '$(System.DefaultWorkingDirectory)/coverage'
    ArtifactName: 'coverage-gap-report'
```

## Troubleshooting

### No Coverage Files Found

**Error:**
```
WARN [Domain] No coverage report found at api-server/tests/NaarNoor.Domain.Tests
```

**Solution:**
1. Ensure tests have been run with coverage collection
2. Check that coverage files exist:
   ```bash
   # Windows
   Get-ChildItem -Path "api-server/tests" -Filter "coverage.cobertura.xml" -Recurse
   
   # Linux/Mac
   find api-server/tests -name "coverage.cobertura.xml"
   ```
3. If missing, run tests with coverage:
   ```bash
   dotnet test --collect:"XPlat Code Coverage"
   ```

### Could Not Parse Coverage

**Error:**
```
WARN [Domain] Could not parse coverage from <path>
```

**Solution:**
1. Verify coverage file is valid XML
2. Check file format matches Cobertura XML format
3. Ensure coverlet.runsettings is configured correctly
4. Regenerate coverage file

### Permission Denied (Shell Script)

**Error:**
```
bash: ./scripts/validate-coverage.sh: Permission denied
```

**Solution:**
```bash
chmod +x ./scripts/validate-coverage.sh
```
